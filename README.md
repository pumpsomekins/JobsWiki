# JobsWiki - 职业生命周期数据平台

一个专注于职业数据分析与可视化的项目，帮助用户了解不同职业的收入潜力、技能发展路径以及AI自动化风险。

## 项目结构

```
JobsWiki/
├── jobs/                    # 职业数据JSON文件
│   ├── chef.json            # 厨师职业数据
│   ├── dasher.json          # 外卖骑手职业数据
│   └── income-types.json    # 收入类型配置
├── chef.html                # 厨师职业展示页面
├── dasher.html              # 外卖骑手职业展示页面
├── _archive/                # 历史版本归档
├── docs/                    # 项目文档
└── .trae/                   # Trae IDE 技能配置
```

## 核心功能

### 1. 职业数据展示
- **收入分析**: 展示不同国家、不同级别的薪资数据
- **职业发展路径**: 清晰的晋升路径和时间预估
- **技能匹配**: AI自动化风险评估与替代方案

### 2. 多维度比较
- **国家对比**: 覆盖美、英、澳、德、日、新、台、中8个地区
- **城市数据**: 各城市薪资与生活成本对比
- **语言要求**: 各地区语言能力要求评估

### 3. 风险评估
- **AI自动化威胁**: 各任务被AI替代的可能性和时间预估
- **职业危害**: 工作环境风险评估
- **生存策略**: 应对自动化的职业发展建议

## 技术特点

- **纯前端实现**: 无需后端，HTML+CSS+JavaScript即可运行
- **响应式设计**: 适配桌面和移动设备
- **数据驱动**: JSON配置，易于扩展新职业

## 使用方法

直接在浏览器中打开HTML文件即可查看：

```bash
# 打开厨师职业页面
open chef.html

# 打开外卖骑手职业页面
open dasher.html
```

## 数据规范

职业数据遵循 `Job Data JSON Specification (v2.0)` 规范，包含：
- `meta`: 职业基本信息和计算配置
- `search`: 搜索关键词和别名
- `countries`: 各国数据
- `levels`: 职业级别定义
- `tasksByLevel`: 各级别任务及AI风险评估
- `insightCards`: 洞察卡片配置

## 扩展指南

如需添加新职业，请参考 `jobs/chef.json` 的结构，并遵循 `.trae/skills/job-data-json-specification-v1-0/SKILL.md` 中的规范。
